# 新崎川 水位 ＋ 周辺雨量ロガー

神奈川県「雨量水位情報」から、**新崎川の水位**と**幕山公園・新崎川まわりの雨量**を
10分間隔で1時間ごとに自動取得し、年間ログに追記します。
GitHub Actions で動くので **自分のPCは起動していなくてOK**。

## 取得する観測所

| 種別 | 観測所 | 場所・備考 |
|---|---|---|
| 水位 | 新崎橋 | 足柄下郡湯河原町中央（新崎川） |
| 雨量 | 南郷山 | 湯河原町南郷山。**幕山公園の真上・新崎川集水域**＝本命 |
| 雨量 | 白銀山 | 湯河原源頭側の高所（約990m）。上流の降り始めを拾う |
| 雨量 | 浅間山 | 幕山隣接。**現在欠測のことあり**（過去に配信停止歴）。不調なら下記から差し替え可 |

新崎川そのものに最も効くのは **南郷山**。白銀山・浅間山は面的な比較用です。

## 基準水位（新崎橋）

| 区分 | 水位 |
|---|---|
| 水防団待機水位 | 0.80 m |
| 氾濫注意水位 | 1.15 m |
| 避難判断水位 | 1.20 m |
| 氾濫危険水位 | 1.65 m |

平常時はおおむね 0.0〜0.1 m 台。

## 出力ファイル（data/）

| ファイル | 内容 |
|---|---|
| `niizaki_waterlevel.csv` | 水位ログ（timestamp / level_m / trend / status / fetched_at） |
| `rainfall.csv` | 雨量ログ・縦持ち（timestamp / station / rain_10min_mm / rain_cum_mm / status / fetched_at） |
| `combined_10min.csv` | **比較用ワイド表**。同時刻で `level_m` と各点の10分雨量を横並び。Excelでそのままグラフ化できます |

時刻はすべて日本時間（ISO8601、例 `2026-06-09T18:30:00+09:00`）。欠測は空欄。重複は自動排除。

## セットアップ（約5分）

1. GitHubで新リポジトリを作成（公開情報なので **Public** 推奨＝Actionsが無制限・止まりにくい）。
2. このフォルダの中身（`niizaki_logger.py` / `.github/workflows/log.yml` / `data/` / `README.md`）をアップロード。
3. **Settings → Actions → General → Workflow permissions** で **「Read and write permissions」** を選んで保存。
4. **Actions** タブ → `log-niizaki-water-level` → **「Run workflow」** で初回手動実行。各CSVに行が増えれば成功。
5. あとは毎時自動で追記されます。

## 観測所の差し替え・追加

`niizaki_logger.py` の先頭付近 `STATIONS = [...]` を編集するだけ。URL末尾の番号で観測所が決まります。
西湘エリアの雨量10分ページ（`.../rain/10/p10102_18_3585_1_8XX.html`）の例：

- 宮上 `_804` / 宮下 `_816`（湯河原寄りの低標高）
- 明神ヶ岳 `_802` / 屏風山 `_803` / 塔ノ峰 `_811`
- 浅間山 `_817`（既定・欠測あり） / 白銀山 `_812` / 南郷山 `_815`

`col` 名（combined表の列名）も合わせて変えてください。

## メモ

- ページに残るのは直近数時間ぶんだけ。1時間ごとの取得で取りこぼしません。
- 10分間隔・4観測所で1年あたり概算20万行弱。CSV/Excelとも問題なし。
- 取得0行や取得失敗はログに警告／エラーが出ます。
